# Todo App - Веб-приложение на Go с REST API и PostgreSQL

## Описание проекта

Это веб-приложение для управления задачами (todo-list), разработанное на языке Go с использованием REST API архитектуры и базы данных PostgreSQL. Проект реализует основные CRUD-операции для работы с задачами и включает систему аутентификации пользователей.

## Функциональность

### Пользовательские возможности:
- Регистрация и вход с JWT-аутентификацией
- Получение списка задач
- Создание новых задач
- Редактирование существующих задач
- Удаление задач

### Безопасность:
- Хеширование паролей
- Middleware для проверки авторизации
- JWT-токены для доступа к API

## Технологический стек

- **Backend**: Golang
- **База данных**: PostgreSQL
- **Архитектура**: REST API
- **Фреймворки и библиотеки**:
    - Gin - HTTP web framework
    - SQLx - работа с базой данных
    - Viper - управление конфигурацией
    - Logrus - логирование
    - JWT - аутентификация
    - Godotenv - переменные окружения

## Структура проекта

```
todo-app/
├── cmd/
│   └── main.go                 # Точка входа в приложение
├── configs/                    # Конфигурационные файлы
├── pkg/
│   ├── handler/                # HTTP-обработчики
│   ├── service/                # Бизнес-логика
│   ├── repository/             # Работа с базой данных
│   └── schema/                 # Схемы данных
├── .env                        # Переменные окружения
├── .gitignore
├── go.mod
├── migrate.exe                 # Миграции базы данных

├── todo.go

```

### Описание структуры:

- **cmd/main.go** - главный файл приложения, точка входа
- **configs/** - конфигурационные файлы приложения
- **pkg/handler/** - обработчики HTTP запросов (контроллеры)
- **pkg/service/** - бизнес-логика приложения
- **pkg/repository/** - слой работы с базой данных
- **pkg/schema/** - модели и схемы данных
- **.env** - файл с переменными окружения
- **go.mod** - файл зависимостей Go модулей
- **migrate.exe** - утилита для миграций базы данных
- **server.go** - настройка и запуск сервера
- **todo.go** - функциональность задач
- **user.go** - функциональность пользователей


## API Endpoints

### Аутентификация:
- `POST /auth/sign-up` - регистрация пользователя
- `POST /auth/sign-in` - вход пользователя

### Работа с задачами:
- `GET /api/lists` - получение списка задач
- `POST /api/lists` - создание новой задачи
- `PUT /api/lists/{id}` - обновление задачи
- `DELETE /api/lists/{id}` - удаление задачи

## Тестирование

Для тестирования API используется **Postman**. В проекте предусмотрены примеры запросов для всех endpoint-ов.

## Примеры запросов

### Регистрация пользователя:
```bash
POST http://localhost:8000/auth/sign-up
Content-Type: application/json

{
  "name": "Daria",
  "username": "daria_user",
  "password": "password123"
}
```

### Вход пользователя:
```bash
POST http://localhost:8000/auth/sign-in
Content-Type: application/json

{
  "username": "daria_user",
  "password": "password123"
}
```

### Создание новой задачи:
```bash
POST http://localhost:8000/api/lists
Content-Type: application/json
Authorization: Bearer {jwt-token}

{
  "title": "Список дел",
  "description": "Написать реферат"
}
```
